# Лабораторная работа 1: Семантическая сегментация водных объектов

## Задание

Проведение исследований с моделями семантической сегментации (задание на четвёрку).  
Фреймворк: `segmentation_models.pytorch`

## Датасет

**Satellite Images of Water Bodies** — коллекция из ~2841 RGB-снимков со спутника Sentinel-2 с бинарными масками сегментации водных объектов.

Практическая задача: автоматический мониторинг водных ресурсов — определение границ водоёмов для экологического мониторинга, контроля засух и предотвращения наводнений.

Источник: https://www.kaggle.com/datasets/franciscoescobar/satellite-images-of-water-bodies

## Установка и запуск

### 1. Создать виртуальное окружение

```bash
python -m venv .venv
```

Активация:

```bash
# Linux / macOS
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Скачать датасет

Скачайте датасет с [Kaggle](https://www.kaggle.com/datasets/franciscoescobar/satellite-images-of-water-bodies) и распакуйте в папку `data/`:

```
data/
├── Images/
│   ├── water_body_1.jpg
│   └── ...
└── Masks/
    ├── water_body_1.jpg
    └── ...
```

### 4. Запустить ноутбук

```bash
cd notebooks
jupyter notebook baseline.ipynb
```

Ноутбук содержит все пункты задания (1–4) и выполняется последовательно сверху вниз.

## Метрики

- **IoU (Jaccard Index)** — основная метрика сегментации, устойчива к дисбалансу классов
- **Dice Coefficient** — F1-аналог для попиксельной классификации
- **Pixel Accuracy** — доля правильно классифицированных пикселей

## Модели и результаты

### Пункт 2 — Бейзлайн

| Модель | Тип | IoU | Dice | Pixel Acc |
|--------|-----|-----|------|-----------|
| Unet + ResNet34 (SMP) | CNN | 0.8064 | 0.8768 | 0.9428 |
| MAnet + MiT-B2 (SMP) | Трансформер | **0.8216** | **0.8858** | **0.9484** |

### Пункт 3 — Проверка гипотез и улучшенный бейзлайн

| Гипотеза | Модель | IoU | Результат |
|----------|--------|-----|-----------|
| H1: аугментации | Unet + ResNet34 + аугм. | 0.8078 | +0.001 к бейзлайну |
| H2: другой энкодер | Unet + EfficientNet-B3 | 0.8059 | на уровне бейзлайна |
| H3: CosineAnnealing LR | Unet + ResNet34 + cosine | 0.7752 | ухудшение |
| **Улучшенный бейзлайн** | Unet + EfficientNet-B3 + аугм. | **0.8117** | +0.005 к базовому Unet |

> Улучшенный бейзлайн обошёл базовый Unet+ResNet34, но не превзошёл MAnet+MiT-B2.

### Пункт 4 — Custom U-Net с нуля

Архитектура: энкодер (4 уровня) → боттлнек → декодер (4 уровня) + skip-connections.  
Каналы: `[32, 64, 128, 256]`.

| Модель | IoU | Dice | Pixel Acc |
|--------|-----|------|-----------|
| Custom U-Net (без улучшений) | 0.7446 | 0.8347 | 0.9176 |
| Custom U-Net (с аугментациями) | 0.7455 | 0.8360 | 0.9187 |

> Custom U-Net уступает SMP-моделям (~0.07 IoU), что ожидаемо: pretrained-энкодеры дают значительное преимущество.
